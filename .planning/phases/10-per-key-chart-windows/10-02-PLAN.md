In `RenderChart()` method, make the following changes:

1. After `var processed = ProcessGaps(_points);` add max value calculation:
   ```csharp
   double maxValue = 100.0;
   foreach (var p in processed)
       if (p.Value > maxValue) maxValue = p.Value;
   // Round up to nearest 10 for clean Y axis (e.g. 137% -> 140%)
   if (maxValue > 100.0)
       maxValue = Math.Ceiling(maxValue / 10.0) * 10.0;
   ```

2. Replace the hardcoded `100.0` in the Y coordinate calculation with `maxValue`:
   Change:
   ```csharp
   double y = PadY + (1.0 - processed[i].Value / 100.0) * (h - 2 * PadY);
   ```
   To:
   ```csharp
   double y = PadY + (1.0 - processed[i].Value / maxValue) * (h - 2 * PadY);
   ```

3. After rendering all segments (after the `foreach (var seg in segments)` loop), draw the 100% reference line if max exceeds 100%:
   ```csharp
   if (maxValue > 100.0)
   {
       double lineY = PadY + (1.0 - 100.0 / maxValue) * (h - 2 * PadY);
       var refLine = new Line
       {
           X1 = PadX,
           Y1 = lineY,
           X2 = w - PadX,
           Y2 = lineY,
           Stroke = new SolidColorBrush(Color.FromArgb(0x80, 0xFF, 0xFF, 0xFF)), // semi-transparent white
           StrokeThickness = 0.5,
           StrokeDashArray = new DoubleCollection { 4, 3 }
       };
       ChartCanvas.Children.Add(refLine);
   }
   ```
   Use `System.Windows.Shapes.Line` (already imported via `using System.Windows.Shapes;`).
   The dashed semi-transparent white line is subtle but visible on the dark chart background.

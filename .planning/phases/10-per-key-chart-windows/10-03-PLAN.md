  <action>
Investigate whether the Anthropic API exposes extra usage spend data. Probe the following:

1. **Check existing rate limit headers more thoroughly:**
   - After a ProbeModelAsync call succeeds, enumerate ALL response headers (not just anthropic-ratelimit-*-utilization).
   - Log/print all header names and values to identify any spend-related headers.
   - Look for patterns like: `anthropic-ratelimit-*-spend`, `anthropic-ratelimit-*-cost`, `x-usage-*`, `x-spend-*`, `anthropic-billing-*`.

2. **Check response body for billing information:**
   - The /v1/messages response body includes a `usage` field with `input_tokens` and `output_tokens`.
   - Check if there are any additional fields like `cost`, `spend`, `billing` in the response JSON.

3. **Try Anthropic Admin API endpoints (if OAuth scope allows):**
   - `GET https://api.anthropic.com/v1/organizations/{org_id}/billing/usage` (speculative)
   - `GET https://api.anthropic.com/v1/billing/usage` (speculative)
   - Use the same OAuth token. Note the response status code and any error message.

4. **Document findings** as a comment block at the top of the research, including:
   - Which headers/endpoints were tried
   - What data is available vs not available
   - Whether spend tracking is feasible from API data
